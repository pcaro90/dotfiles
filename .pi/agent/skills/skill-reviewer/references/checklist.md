# Skill Review Checklist

Use this checklist as a quick pass after the structured review.

- [ ] `name` follows naming conventions
- [ ] `description` is third person, intent-focused, under 1024 characters
- [ ] Body uses second-person imperative throughout
- [ ] No third-person self-reference (for example, "the agent should")
- [ ] Every step has success criteria
- [ ] No unnecessary explanations of concepts the agent already knows
- [ ] Behavioral rules follow prohibition + reason + alternative
- [ ] Persona is functional, not personality-based
- [ ] Specificity matches operation fragility
- [ ] Edge cases and error handling are addressed
- [ ] No agent-name-specific references unless necessary
- [ ] No repeated information across sections
- [ ] Progressive disclosure is respected: lean `SKILL.md`, references offloaded
- [ ] Output formats use exact templates
- [ ] Input/Output examples are included where applicable
- [ ] Writing explains why, not just what
- [ ] Description is pushy enough to combat undertriggering
- [ ] Test cases are included for objective-output skills
- [ ] No misleading or surprising behavior
